# MySQL 数据分析练习项目总结

## 1. 项目背景

本项目是从数据集到语法学习再到综合应用的基础mysql数据分析业务的技能学习，主要是通过用户表，订单表，商品表，三个数据集，通过学习基础的查询再到多表连接和条件查询等综合查询来解决业务查询问题的整个项目

## 2. 数据表说明

本项目主要使用三张数据表：

| 表名 | 含义 | 主要内容 |
|---|---|---|
| users | 用户表 | 用户编号、姓名、年龄、城市、注册日期、会员等级 |
| products | 商品表 | 商品编号、商品名称、商品分类、价格、供应商、上架日期 |
| orders | 订单表 | 订单编号、用户编号、商品编号、订单日期、数量、单价、优惠金额、支付方式、订单状态 |

三张表之间的关系是：

```sql
orders.user_id = users.user_id
orders.product_id = products.product_id
```

也就是说：

- 一个订单属于某一个用户；
- 一个订单对应某一个商品；
- 订单表是连接用户表和商品表的核心表，订单表是作为用户表和商品表连接的核心桥梁，可以达到三表连接 

## 3. 学习内容总结

| 学习阶段 | 核心内容 | 作用 |

| 第 8 天 | LEFT JOIN、RIGHT JOIN、NULL | 保留主表数据，处理没有匹配的数据 |
| 第 9 天 | 子查询 | 把一个查询结果作为另一个查询的条件或临时表 |
| 第 10 天 | CASE WHEN | 根据条件生成分类标签 |
| 第 11 天 | 日期函数、文本函数、数字函数 | 对日期、字符串和数字进行处理 |
| 第 12 天 | 订单分析综合练习 | 围绕订单数据做综合分析 |
| 第 13 天 | 用户与商品分析综合练习 | 围绕用户和商品维度做综合分析 |
| 第 14 天 | 项目总结与结构整理 | 把零散练习整理成项目形式 |

## 5. 项目分析内容

### 5.1 订单分析

订单分析主要关注订单数量、订单金额、订单状态、支付方式和订单时间等问题。

常见分析问题包括：

- 总订单数量是多少；
- 不同订单状态下分别有多少订单；
- Paid 订单的实付金额是多少；
- 每个月的订单数量和销售金额是多少；
- 不同支付方式的订单数量和平均实付金额是多少。

这一部分主要训练：

- 聚合函数；
- GROUP BY 分组；
- WHERE 条件筛选；
- 日期函数；
- 计算字段。

### 5.2 用户分析

用户分析主要关注用户所在城市、会员等级、下单情况和消费金额等问题。

常见分析问题包括：

- 每个城市有多少用户；
- 每个会员等级有多少用户；
- 每个用户下了多少订单；
- 哪些用户没有下过订单；
- 每个城市的 Paid 订单实付金额是多少。

这一部分主要训练：

- LEFT JOIN；
- COUNT；
- SUM；
- GROUP BY；
- NULL 判断；
- 用户表和订单表的关联分析。

### 5.3 商品分析

商品分析主要关注商品分类、商品价格、购买数量和商品销售额等问题。

常见分析问题包括：

- 每个商品分类有多少商品；
- 每个商品分类的购买总件数是多少；
- 每个商品的实付总金额是多少；
- 哪些商品没有被购买过；
- 不同商品分类的 Paid 订单销售额是多少。

这一部分主要训练：

- 商品表和订单表关联；
- 分类统计；
- 聚合计算；
- LEFT JOIN 查找未购买商品；
- 多维度分析。

## 6. 当前阶段收获

通过这个阶段的学习，我对 SQL 有了更清晰的理解。

SQL 不只是写查询语句，而是通过表和字段去回答业务问题。面对一个问题时，我需要先判断：

1. 需要查询哪张表；
2. 是否需要关联其他表；
3. 查询结果需要显示哪些字段；
4. 是否需要筛选条件；
5. 是否需要分组统计；
6. 是否需要对分组结果继续筛选；
7. 是否需要排序、计算字段或条件判断。

现在我已经能够使用 MySQL 完成一些基础的数据分析任务，例如订单统计、用户分析、商品分析和多表关联查询。

## 7. 目前还需要继续加强的地方

虽然已经掌握了 MySQL 的基础查询能力，但还有一些地方需要继续练习：

- 多表连接时，先判断主表和从表的能力还需要加强；
- LEFT JOIN 中 NULL 的理解需要继续巩固；
- GROUP BY 后字段选择的规范性还需要提高；
- 子查询和 JOIN 的使用场景还需要多做对比；
- 查询语句的书写格式需要继续保持规范。
